eAplicação base com locais e percursos em que cada percurso tem parâmetros de:
-pavimento
-inclinação
-qualidade do ambiente e do ar (poluição sonora, visual, indice de carbono, polen/alergias...)
-trânsito
-multidão
-hora do dia
-passadeiras
-passeios
-acessibiliadde a cadeira de rodas, pessoas com limitações
(mais algumas coisas)

Possíveis locais nos mapas:
-locais de construção/obras (obstrução de vias, poluição..)
-hospitais
-parques naturais
-fábricas
-escolas
-áreas comerciais
-áreas rurais
-rios/mares/lagos
-multibanco
-centro de camionagem/táxis/...
-paragem de autocarros
-esquadra da polícia
-bombeiros
-farmácia
-camara municipal/junta de freguesia
-ecopontos
-estações de carregamento de carros elétricos
- bebedouros, ...

Coisas extra:
-opção de login/sign in
-sistema de bicicletas ou trotinetes (+ sustentabilidade)
-banco de cidades
-"uber"/transportes
-imposições de horário (limitações no horário) 
-luz
-metereologia
-hora do dia
-distância percorrida (dependendo do meio de transporte, sistema de pontos para obter vantagens promocionais)

Informações e inputs do usuário (selecionar entre opcções: "#")
-nome
-idade
-incapacidades #
-problemas de saúde #
-alergias #
-meio de transporte #
-destino #

Métricas, cálculos:
-tempo que demora a percorrer caminho
-alternativas de rota limitadas (função do tempo, das condições, das preferências... usuário escolhe)

Interface:
- interativa, estilo aplicação de telmóvel

main.py -- parte que interliga todos os outros ficheiros

modelo\ classes a usar
local.py -- hospitais, parques, escolas, fábricas, etc.
percurso.py -- dados que um percurso pode ter (origem, destino, indicadores de poulição,etc.)
indicadores_ambientais.py -- fatores de poluição do ambiente e fatores que beneficiam o ambiente
indicadores_acessibilidade.py -- fatores que dificultam/facilitam o trânsito do usuario
utilizador.py -- dados do utilizador (nome, idade, problemas de saude, etc.)
grafo.py -- rede de percursos com metodos para adicionar nós (locais) e arestas (ligações entre lugares), calcular percursos, etc.

pelo que estive a ver, é boa prática ter um ficheiro para uma única classe

servicos\ funções do sistema
gestor_dados.py -- gere os ficheiros necessários (carregar, guardar) e permite mexer na base de dados
motor_analise.py -- no geral, trata dos cálculos de inidces de conforto e de outras formulas
motor_recomendacao.py -- encontra caminhos e adapta-os ao perfil do utilizador

interface\ o que o utilizador vê/interage
consola.py -- loop principal, mostrar menus, interpretar comandos
comandos.py -- implementa as funções ligadas à interação do utilizador (guardar, sair, recomendar, etc.)
gui.py -- parte gráfica

dados\ possíveis bases de dados

docs\ possíveis documentos a guardar, como o README ou exemplos de outputs
